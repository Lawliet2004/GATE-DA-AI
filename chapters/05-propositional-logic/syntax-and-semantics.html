Syntax and Semantics of Propositional Logic

GATE DA — High Importance

Prerequisites

  • Basic understanding of mathematical logic
  • Familiarity with Boolean algebra concepts

🎯 Learning Objectives

  • Understand the syntax of propositional logic: atomic propositions and connectives
  • Define well-formed formulas (WFFs) formally
  • Apply operator precedence to parse complex formulas
  • Evaluate formulas under given truth assignments

1. What is Propositional Logic?

Propositional logic (also called sentential logic or zeroth-order logic) is the simplest form of formal logic. It deals with propositions—statements that are either true or false—and how they can be combined using logical connectives.

Definition: Proposition

A proposition is a declarative sentence that is either true (⊤) or false (⊥), but not both. It has a definite truth value.

Examples of Propositions

  • "The Earth revolves around the Sun." — True
  • "7 + 5 = 13" — False
  • "GATE DA 2026 exam has an AI section." — True

Non-Propositions (Not Declarative)

  • "What time is it?" — Question, no truth value
  • "Close the door." — Command, no truth value
  • "x + 2 = 5" — Open sentence (depends on x)

The syntax defines the alphabet (symbols) and grammar (rules for forming valid expressions) of the language.

2.2 The Five Logical Connectives

Negation (¬)

¬P is true when P is false, and false when P is true. It flips the truth value.

¬P is true ⟺ P is false

Conjunction (∧)

P ∧ Q is true only when both P and Q are true.

P ∧ Q is true ⟺ P is true AND Q is true

Disjunction (∨)

P ∨ Q is true when at least one of P or Q is true. This is inclusive OR.

P ∨ Q is false ⟺ P is false AND Q is false

Implication (→)

P → Q is false only when P is true and Q is false. P is the antecedent (hypothesis), Q is the consequent (conclusion).

P → Q is false ⟺ P is true AND Q is false

⚠️ Common GATE Mistake: Implication

Many students get confused: P → Q is true when P is false, regardless of Q. A false hypothesis makes the entire implication vacuously true. For example: "If 2 + 2 = 5, then the moon is made of cheese" is True!

Biconditional (↔)

P ↔ Q is true when P and Q have the same truth value.

P ↔ Q is true ⟺ (P → Q) ∧ (Q → P)

3. Well-Formed Formulas (WFFs)

Definition: Well-Formed Formula (WFF)

A well-formed formula is defined recursively:

  1. Base case: Every propositional variable (P, Q, R, ...) is a WFF. The constants ⊤ and ⊥ are WFFs.
  2. Negation: If α is a WFF, then ¬α is a WFF.
  3. Binary connective: If α and β are WFFs, then (α ∧ β), (α ∨ β), (α → β), and (α ↔ β) are WFFs.
  4. Nothing else is a WFF.

WFF or Not?

ExpressionWFF?Reason
P ∧ Q✅ YesBinary connective applied to two propositional variables
¬(P → Q)✅ YesNegation applied to an implication WFF
∧ P Q❌ NoConnective must be between two WFFs (infix notation)
P ∧ ∨ Q❌ NoTwo consecutive binary connectives without operand
((P ∨ Q) ∧ ¬R) → S✅ YesValid recursive construction

4. Operator Precedence

To reduce the number of parentheses, we adopt a standard precedence (from highest to lowest):

Operator Precedence (Highest to Lowest)

PriorityOperatorNameAssociativity
1 (Highest)¬NegationRight
2∧ConjunctionLeft
3∨DisjunctionLeft
4→ImplicationRight
5 (Lowest)↔BiconditionalLeft

📝 Key Point for GATE

Implication (→) is right-associative: P → Q → R means P → (Q → R), not (P → Q) → R.

Example 1: Parsing P ∧ Q ∨ R → S

Step 1: ∧ binds tighter than ∨: (P ∧ Q) ∨ R → S

Step 2: ∨ binds tighter than →: ((P ∧ Q) ∨ R) → S

Fully parenthesized: ((P ∧ Q) ∨ R) → S

Example 2: Parsing ¬P ∧ Q → R ∨ S

Step 1: ¬ has highest precedence: (¬P) ∧ Q → R ∨ S

Step 2: ∧ next: ((¬P) ∧ Q) → R ∨ S

Step 3: ∨ next: ((¬P) ∧ Q) → (R ∨ S)

Fully parenthesized: ((¬P) ∧ Q) → (R ∨ S)

Example 3: Parsing P → Q → R ↔ S ∨ ¬T

Step 1: ¬ first: P → Q → R ↔ S ∨ (¬T)

Step 2: ∨ binds: P → Q → R ↔ (S ∨ (¬T))

Step 3: → is right-associative: P → (Q → R) ↔ (S ∨ (¬T))

Step 4: → again: (P → (Q → R)) ↔ (S ∨ (¬T))

Fully parenthesized: (P → (Q → R)) ↔ (S ∨ (¬T))

5. Semantics: Truth Assignments and Interpretations

Definition: Truth Assignment (Interpretation)

A truth assignment (also called an interpretation or valuation) is a function v that maps every propositional variable to a truth value:

v : {P, Q, R, ...} → {True, False}

The truth value of any compound formula is then determined by the truth values of its components and the semantics of the connectives.

Semantic Evaluation Rules

Given an interpretation v, the truth value of a formula α, denoted v(α), is:

  • v(¬α) = True iff v(α) = False
  • v(α ∧ β) = True iff v(α) = True and v(β) = True
  • v(α ∨ β) = True iff v(α) = True or v(β) = True
  • v(α → β) = True iff v(α) = False or v(β) = True
  • v(α ↔ β) = True iff v(α) = v(β)

Example: Evaluate (P → Q) ∧ ¬R under v(P)=T, v(Q)=F, v(R)=T

  1. v(P → Q) = v(P)=T and v(Q)=F, so P → Q = False (T → F = F)
  2. v(¬R) = ¬True = False
  3. v((P → Q) ∧ ¬R) = False ∧ False = False

📝 Number of Possible Interpretations

For a formula with n distinct propositional variables, there are 2n possible truth assignments. This is why truth tables have 2n rows.

7. GATE Exam Tips

🎯 Frequently Tested Concepts

  • Operator precedence — Questions often test whether you can parse formulas correctly without full parenthesization.
  • Implication truth table — The fact that F → T = T and F → F = T trips up many students.
  • WFF identification — Distinguish valid formulas from syntactically incorrect expressions.
  • Number of interpretations — For n variables, there are exactly 2n interpretations.
