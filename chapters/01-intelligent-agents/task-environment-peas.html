Task Environments & the PEAS Framework

GATE Relevance: High

Prerequisites

  • Agents & Environments — Agent definition, sensors, actuators
  • Rationality — Performance measure concept

🎯 Learning Objectives

  • Apply the PEAS framework (Performance, Environment, Actuators, Sensors) to any agent
  • Classify environments using six properties: Observable, Deterministic, Episodic, Static, Discrete, Single/Multi-agent
  • Analyze real-world systems using PEAS and environment properties
  • Identify how environment properties influence agent design
  • Solve GATE questions on task environment classification

1. The PEAS Framework

Before designing an intelligent agent, we must fully specify its task environment. The PEAS framework provides a systematic way to do this.

PEAS

PEAS stands for:

  • Performance measure — How success is evaluated
  • Environment — The external world the agent operates in
  • Actuators — How the agent affects the environment
  • Sensors — How the agent perceives the environment

Specifying PEAS completely defines the task environment—the “problem” to which the agent is the “solution.” Different task environments require different agent designs.

1.1 PEAS Analysis: Automated Taxi Driver

ComponentDescription
PerformanceSafely reaching the destination, minimizing travel time, obeying traffic laws, passenger comfort, maximizing fare revenue
EnvironmentRoads, other vehicles, pedestrians, traffic signals, weather, passengers, road signs, construction zones
ActuatorsSteering wheel, accelerator, brakes, turn signals, horn, display screen (for passenger communication)
SensorsCameras (front, rear, side), LIDAR, radar, GPS, speedometer, odometer, accelerometer, microphone, engine sensors

1.2 PEAS Analysis: Medical Diagnosis System

ComponentDescription
PerformanceCorrect diagnosis (accuracy), patient health outcomes, minimizing unnecessary tests, cost-effectiveness
EnvironmentPatient population, hospital/clinic, available tests and treatments, disease prevalence, medical knowledge base
ActuatorsDisplay diagnosis on screen, recommend tests, suggest treatments, generate reports, send alerts
SensorsKeyboard/form input (symptoms), electronic health records, lab test results, medical imaging data, patient questionnaires

1.3 PEAS Analysis: Satellite Image Analysis System

ComponentDescription
PerformanceClassification accuracy, coverage of area, speed of processing, false positive/negative rates
EnvironmentSatellite imagery, geographic regions, atmospheric conditions, different land types, time of day/year
ActuatorsDisplay annotated images, output classification labels, generate maps, send notifications for anomalies
SensorsSatellite image feed (multispectral, infrared, optical), metadata (coordinates, timestamp, resolution), weather data

2. Properties of Task Environments

Russell & Norvig identify six key properties that characterize any task environment. These properties are critical for GATE because they determine which agent architecture is appropriate.

Mnemonic: ODESSA

Remember the six properties with ODESSA:
Observable • Deterministic • Episodic • Static • diScrete • Agents (single/multi)

2.1 Fully Observable vs. Partially Observable

Fully Observable

An environment is fully observable if the agent’s sensors give it access to the complete state of the environment at each point in time. The agent does not need to maintain any internal state (memory) to keep track of the world.

Partially Observable

An environment is partially observable if the agent cannot see the entire state—due to noisy sensors, inaccurate sensors, or parts of the state that are simply not detectable. The agent needs to maintain internal state (a model of the world) to track unobserved aspects.

Unobservable

If the agent has no sensors at all, the environment is unobservable. The agent must act based solely on prior knowledge.

Examples:

  • Fully observable: Chess (both players can see the entire board)
  • Partially observable: Poker (you cannot see opponents’ cards), driving (you cannot see around corners)
  • Unobservable: A robot navigating in complete darkness without any sensors

2.2 Deterministic vs. Stochastic

Deterministic

An environment is deterministic if the next state is completely determined by the current state and the agent’s action. There is no randomness.

Stochastic

An environment is stochastic if there is randomness or uncertainty in the outcome of actions. The next state is partly determined by random factors.

Nondeterministic

Sometimes called nondeterministic when outcomes are uncertain but not described by explicit probabilities (as opposed to stochastic, which implies probabilistic modeling).

Note: If the environment is partially observable, it may appear stochastic to the agent even if it is actually deterministic (because the agent cannot see all the relevant state). This is called being strategic in some texts.

Examples:

  • Deterministic: Chess, vacuum cleaner world (without random dirt)
  • Stochastic: Backgammon (dice rolls), poker, real-world driving (other drivers’ actions are unpredictable)

2.3 Episodic vs. Sequential

Episodic

An environment is episodic if the agent’s experience is divided into independent episodes. Each episode consists of perceiving and then performing a single action. The action in one episode does not affect subsequent episodes. The agent does not need to plan ahead.

Sequential

An environment is sequential if the current decision can affect all future decisions. The agent must think ahead because actions have long-term consequences. Sequential environments require planning.

Examples:

  • Episodic: Image classification (classifying one image doesn’t affect the next), spam detection (each email is independent), quality inspection on an assembly line
  • Sequential: Chess (each move affects future moves), driving (steering affects future position), medical treatment (treatment affects future health state)

2.4 Static vs. Dynamic

Static

An environment is static if it does not change while the agent is deliberating (deciding what to do). The agent can take as long as it wants to make a decision without the world changing.

Dynamic

An environment is dynamic if it can change while the agent is thinking. The agent must decide quickly or accept that the world may have changed by the time it acts.

Semidynamic

An environment is semidynamic if the environment itself does not change with time, but the agent’s performance score changes. Example: timed chess—the board doesn’t change while you think, but your clock is ticking.

Examples:

  • Static: Crossword puzzles, solved-state Rubik’s cube analysis
  • Semidynamic: Chess with a time clock, timed exams
  • Dynamic: Driving (traffic moves), real-time strategy games, stock trading

2.5 Discrete vs. Continuous

Discrete

An environment is discrete if there are a finite (countable) number of distinct states, percepts, and actions. The state space is countable.

Continuous

An environment is continuous if states, percepts, or actions vary continuously (e.g., real-valued sensor readings, continuous steering angles).

Examples:

  • Discrete: Chess (finite board positions), tic-tac-toe, vacuum world
  • Continuous: Driving (continuous steering angle, speed, position), robot navigation in physical space, voice recognition (continuous audio signals)

2.6 Single-Agent vs. Multi-Agent

Single-Agent

An environment with only one agent operating. The agent doesn’t need to model other agents’ behavior.

Multi-Agent

An environment with multiple agents. Multi-agent environments can be:

  • Competitive: Agents have opposing goals (e.g., chess, Go)
  • Cooperative: Agents work toward a shared goal (e.g., robot swarms)
  • Mixed: Some competition and some cooperation (e.g., traffic—everyone wants to reach their destination, but must cooperate to avoid accidents)

Examples:

  • Single-agent: Crossword puzzle, solo Sudoku
  • Multi-agent competitive: Chess, poker, Go
  • Multi-agent cooperative: Robot soccer team, warehouse robot fleet
  • Multi-agent mixed: Autonomous driving (other cars are agents)

GATE Tip

The most commonly asked environment properties in GATE are Observable and Deterministic. Expect questions like: “Which of the following environments is partially observable and stochastic?” Always justify your classification by identifying what information the agent cannot see (for partial observability) and what randomness exists (for stochasticity).

The hardest environments (partially observable, stochastic, sequential, dynamic, continuous, multi-agent) require the most complex agent designs. Real-world environments like autonomous driving exhibit all these “hard” properties simultaneously.

5. Worked Examples

Worked Example 1: Classifying an Online Shopping Recommendation System

Question: Perform a complete PEAS analysis and environment classification for an online shopping recommendation system (like Amazon’s “Customers who bought this also bought”).

Solution:

PEAS:

  • Performance: Click-through rate, purchase conversion rate, customer satisfaction, revenue generated
  • Environment: Users, product catalog, purchase history, browsing history, reviews, pricing, competitors
  • Actuators: Display recommended products, adjust ranking of suggestions, send notification emails
  • Sensors: User click data, purchase logs, browsing history, search queries, ratings/reviews, session duration

Environment classification:

  1. Partially observable: The system cannot see the user’s true preferences, mood, or intent—only their behavior.
  2. Stochastic: User behavior is unpredictable and influenced by many external factors.
  3. Sequential: Recommendations affect what the user sees next, which affects future clicks and purchases.
  4. Dynamic: New products are added, prices change, user preferences evolve.
  5. Discrete: Products, users, and actions (click/buy/ignore) are countable.
  6. Multi-agent: Other users influence recommendations (collaborative filtering); competitors offer alternatives.
Worked Example 2: Classifying a Crossword Puzzle Agent

Question: Classify the environment for an AI that solves crossword puzzles.

Solution:

  1. Fully observable: The agent can see all the clues and the current state of the grid.
  2. Deterministic: Filling in a letter produces a known, certain result.
  3. Sequential: Filling one word constrains other words (shared letters).
  4. Static: The puzzle doesn’t change while the agent thinks.
  5. Discrete: Finite grid cells, finite alphabet of letters.
  6. Single-agent: Only one solver.

Classification: Fully observable, Deterministic, Sequential, Static, Discrete, Single-agent.

Worked Example 3: Real-Time Stock Trading Bot

Question: A high-frequency trading bot buys and sells stocks milliseconds apart. Classify its environment.

Solution:

  1. Partially observable: The bot sees market data (prices, volumes) but not other traders’ strategies, hidden orders, or upcoming news events.
  2. Stochastic: Market movements are inherently unpredictable and influenced by countless factors.
  3. Sequential: Each trade affects the portfolio, which influences future trading decisions.
  4. Dynamic: Prices change continuously, even while the bot is computing.
  5. Continuous: Prices are real-valued; time is continuous; trade quantities can be fractional.
  6. Multi-agent: Thousands of other trading bots and human traders operate simultaneously.

Classification: Partially observable, Stochastic, Sequential, Dynamic, Continuous, Multi-agent — this is one of the hardest possible environment configurations.

6. GATE-Style Questions

GATE Practice Question 1 (1 mark)

Question: The game of chess (without a clock) is best described as:

(A) Partially observable, stochastic, sequential
(B) Fully observable, deterministic, sequential
(C) Fully observable, stochastic, episodic
(D) Partially observable, deterministic, episodic

Answer: (B). Both players can see the entire board (fully observable), moves produce deterministic outcomes (deterministic), and each move affects future moves (sequential).

GATE Practice Question 2 (2 marks)

Question: Consider a robot vacuum cleaner operating in a large office building during working hours. People may move furniture, spill coffee, or close doors. Classify this environment on all six PEAS environment properties. Justify each.

Answer:

  • Partially observable: The robot cannot see all rooms simultaneously; it has limited sensor range.
  • Stochastic: People spilling coffee or moving furniture introduces randomness.
  • Sequential: Cleaning one room affects where to go next; dirt accumulates over time.
  • Dynamic: The environment changes while the robot cleans (people move things, doors open/close).
  • Continuous: The robot moves in continuous physical space with continuous sensor readings.
  • Multi-agent: People are effectively other agents that modify the environment.

7. Common Mistakes & Misconceptions

Mistake 1: Confusing Stochastic with Partially Observable

These are different properties. A partially observable environment means the agent can’t see everything. A stochastic environment means actions have random outcomes. Chess is fully observable but deterministic. Poker is partially observable AND stochastic. An environment can be partially observable but deterministic (e.g., a robot in a dark room with no randomness).

Mistake 2: Calling Chess “Dynamic”

Chess (without a clock) is static, not dynamic. The board doesn’t change while you’re thinking. The opponent waits for your move. With a clock, it’s semidynamic (the clock ticks, affecting performance, but the board itself doesn’t change).

Mistake 3: Episodic vs. Sequential Confusion

If current actions affect future states, the environment is sequential. If each decision is independent, it’s episodic. In spam detection, classifying one email usually doesn’t affect the next (episodic). In chess, each move changes the future (sequential). When in doubt, ask: “Does the current action matter for future decisions?”

📚 Summary

  • PEAS (Performance, Environment, Actuators, Sensors) fully specifies a task environment.
  • Six environment properties (ODESSA): Observable, Deterministic, Episodic, Static, Discrete, Agents.
  • Fully observable = agent sees all state; partially observable = agent has incomplete information.
  • Deterministic = no randomness; stochastic = uncertainty in outcomes.
  • Episodic = independent decisions; sequential = decisions affect future.
  • Static = environment doesn’t change during deliberation; dynamic = environment changes in real-time.
  • Discrete = countable states/actions; continuous = real-valued states/actions.
  • Single-agent = one agent; multi-agent = competitive, cooperative, or mixed.
  • The hardest environments combine all the “difficult” properties (partially observable, stochastic, sequential, dynamic, continuous, multi-agent).
