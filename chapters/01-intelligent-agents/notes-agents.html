📝 Quick-Revision Notes: Intelligent Agents

GATE Relevance: High

1. Key Definitions (Numbered)

  1. Agent: An entity that perceives its environment through sensors and acts upon it through actuators.
  2. Percept: The agent’s sensory input at a single instant.
  3. Percept sequence: The complete history of all percepts ever received.
  4. Agent function: A mathematical mapping from every possible percept sequence to an action: f : P* → A.
  5. Agent program: A concrete software/hardware implementation of the agent function (runs on an architecture).
  6. Rational agent: Selects the action that maximizes expected performance, given its percept sequence and built-in knowledge.
  7. Omniscient agent: Knows the actual outcome of every action—impossible in practice; distinct from rational.
  8. Performance measure: An external criterion for evaluating how successful an agent is in its environment.
  9. Task environment: The complete specification: Performance measure, Environment, Actuators, Sensors (PEAS).
  10. Autonomy: An agent is autonomous to the extent that its behavior depends on its own experience rather than solely on built-in knowledge.

3. Environment Properties — ODESSA Mnemonic

💡 Mnemonic: ODESSA

Observable • Deterministic • Episodic • Static • diScrete • Agents

“Old Dogs Eat Sausages, Steaks, and Apples”

Property Values What It Means Example (Fully) Example (Partially/No)
Observable Fully / Partially / Unobservable Can agent perceive entire environment state? Chess (board visible) Poker (hidden cards)
Deterministic Deterministic / Stochastic / Non-deterministic Is next state fully determined by current state + action? Chess Backgammon (dice)
Episodic Episodic / Sequential Is each action independent, or do earlier actions affect later ones? Image classification Chess (whole game)
Static Static / Dynamic / Semidynamic Does environment change while agent deliberates? Crossword puzzle Self-driving car (dynamic)
Discrete Discrete / Continuous Are states, time, percepts, and actions countable? Chess (finite states) Robot navigation (continuous)
Agents Single-agent / Multi-agent (competitive/cooperative) How many agents interact in the environment? Crossword (single) Poker (competitive multi)
Stochastic vs. Non-deterministic

Stochastic: Outcomes have quantified probabilities (e.g., dice roll: P(6)=1/6).
Non-deterministic: Multiple outcomes are possible but no probabilities are assigned.

Semidynamic Environments

Environment doesn’t change, but the agent’s performance score changes with time. Example: Chess with a clock (board is static, but time ticks).

5. Agent Types Comparison

Agent Type State Maintained Decision Mechanism Limitations Example
Simple Reflex None Condition → Action rules Fails in partially observable envs; no memory Thermostat
Model-Based Reflex Internal model of world Rules + internal state Model may be inaccurate Anti-lock brakes
Goal-Based Model + goal Search / planning No trade-offs; binary goal (achieved or not) GPS navigation
Utility-Based Model + utility function Maximize expected utility Complex; utility function design is hard Self-driving car
Learning Model + learning element Improve via experience (critic feedback) Initial performance may be low AlphaGo, spam filter
Hierarchy Reminder

Simple Reflex ⊂ Model-Based ⊂ Goal-Based ⊂ Utility-Based ⊂ Learning Agent. Each type adds capabilities over the previous one.

6. Four Factors of Rationality

⚡ Quick Facts for GATE

  1. Performance measure — defines success
  2. Prior knowledge of the environment
  3. Available actions the agent can perform
  4. Percept sequence received so far

A rational agent selects the action that maximizes expected performance, given factors 1–4.

⚠ Common GATE Trap

Rational ≠ Omniscient. A rational agent makes the best decision given available information. It can still fail if the outcome is unpredictable. Rationality requires information gathering (exploration) when uncertain.

7. Key Formulas & Concepts

  • Agent function: f : P* → A
  • Agent program = Agent function + Architecture
  • Table-driven agent: Stores all |P*| entries. For n percept values over t time steps: |P*| = ∑ ni for i=1..t — grows exponentially, hence impractical.
  • Performance score (vacuum world): +1 per clean square per time step.
  • Transition model (model-based): state′ = f(state, action)
  • Sensor model: Maps true world state to percept the agent receives
  • Utility function: U : States → ℜ — maps states to real-valued happiness scores
  • Expected utility: EU(a) = ∑s′ P(s′|a) ⋅ U(s′)

8. GATE Quick Facts

🎯 Most Frequently Tested Concepts

  • Classifying environments using the 6 ODESSA properties
  • Identifying the correct agent type for a given scenario
  • Distinguishing rational vs. omniscient agents
  • PEAS analysis for a described system
  • Why table-driven agents are impractical (exponential storage)
  • Model-based vs. simple reflex in partially observable environments
  • Learning agent components: performance element, critic, learning element, problem generator
⚠ Common Mistakes in GATE
  • Confusing “stochastic” with “non-deterministic”—stochastic has quantified probabilities
  • Saying chess is “static” when played with a clock (it’s semidynamic)
  • Calling a spam filter “sequential”—each email is classified independently (“episodic”)
  • Thinking goal-based agents have utility functions (they only have binary goals)
  • Forgetting that simple reflex agents cannot handle partial observability

10. One-Page Summary

  • An agent perceives via sensors, acts via actuators, and is described by an agent function f : P* → A.
  • A rational agent maximizes expected performance given its percept sequence—it is NOT omniscient.
  • Specify any AI problem using PEAS: Performance, Environment, Actuators, Sensors.
  • Classify environments with ODESSA: Observable, Deterministic, Episodic, Static, diScrete, Agents.
  • Five agent types form a hierarchy: Simple Reflex → Model-Based → Goal-Based → Utility-Based → Learning.
  • Simple reflex agents fail under partial observability; model-based agents add internal state.
  • Goal-based agents use search/planning; utility-based agents handle trade-offs with a utility function.
  • Learning agents continually improve through the critic → learning element → performance element feedback loop.
