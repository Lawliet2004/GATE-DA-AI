Supervised Learning — Fundamentals

GATE Relevance: High

Prerequisites

  • Probability Basics — conditional probability, Bayes’ theorem
  • Basic linear algebra — vectors, matrices, matrix multiplication
  • Calculus — derivatives, partial derivatives, chain rule

🎯 Learning Objectives

  • Distinguish between classification and regression tasks
  • Understand training, validation, and test set splits
  • Define hypothesis space and inductive bias
  • Derive and compare common loss functions
  • Formulate the Empirical Risk Minimization (ERM) framework

1. What is Supervised Learning?

Definition — Supervised Learning

Given a training set D = {(x1, y1), (x2, y2), …, (xn, yn)} where each xi ∈ ℜd is a feature vector and yi is the corresponding label, the goal is to learn a function h: X → Y that generalises well to unseen data.

In supervised learning the “supervisor” provides the correct answer (label) for every training example. The learner’s job is to find a mapping from inputs to outputs that works well on new, unseen examples drawn from the same distribution.

📝 Key Terminology

  • Feature vector (x): A numerical representation of the input, e.g., [height, weight, age].
  • Label (y): The output we want to predict — a class label (classification) or a real number (regression).
  • Training example: A single (x, y) pair used to train the model.
  • Instance space (X): The set of all possible input vectors.

2. Classification vs Regression

AspectClassificationRegression
Output typeDiscrete class label (e.g., {0,1}, {cat, dog})Continuous real value (e.g., price, temperature)
ExamplesSpam detection, image recognition, disease diagnosisHouse price prediction, stock forecasting, age estimation
Loss functions0-1 loss, cross-entropy, hinge lossSquared error, absolute error, Huber loss
EvaluationAccuracy, Precision, Recall, F1, AUCMSE, RMSE, MAE, R²
Decision boundarySeparates classes in feature spaceFits a curve/surface through data points

💡 Example — Classification vs Regression

Classification: Given patient data (age=55, cholesterol=240, BP=150), predict whether the patient has heart disease: y ∈ {Yes, No}.

Regression: Given the same patient data, predict the patient’s expected medical cost: y ∈ ℜ (e.g., ₹45,000).

3. Training, Validation, and Test Sets

Proper data splitting is essential to estimate how well a model will generalise to unseen data.

Standard Split Strategy

  • Training set (60–80%): Used to fit model parameters (learn weights).
  • Validation set (10–20%): Used for hyperparameter tuning and model selection. Not used for training.
  • Test set (10–20%): Used only once at the very end for final performance evaluation.

⚠ Common GATE Trap

The test set must never be used during training or model selection. Using test data for hyperparameter tuning leads to data leakage and over-optimistic performance estimates. GATE questions sometimes test whether candidates understand this distinction.

💡 Example — Data Split

Dataset of 1000 samples: Training = 700, Validation = 150, Test = 150.

We train multiple models (e.g., different polynomial degrees) on the 700 samples, evaluate each on the 150 validation samples, pick the best, then report its score on the 150 test samples.

4. Hypothesis Space and Inductive Bias

Definition — Hypothesis Space (H)

The hypothesis space H is the set of all possible functions h: X → Y that the learning algorithm considers. For example, if we use linear models in 2D, then H = {h(x) = w0 + w1x1 + w2x2 | wi ∈ ℜ}.

Definition — Inductive Bias

The set of assumptions a learning algorithm makes to generalise beyond the training data. Without inductive bias, a learner cannot generalise at all (No Free Lunch Theorem).

AlgorithmInductive Bias
Linear RegressionThe target function is linear in the features
k-NN (k=1)Similar inputs have similar outputs; locality
Decision TreesShort trees are preferred (Occam’s Razor)
Naïve BayesFeatures are conditionally independent given class
SVMMaximum margin classifier; large margin implies better generalisation

No Free Lunch Theorem (Wolpert, 1996)

There is no single learning algorithm that works best for every problem. Averaged over all possible data distributions, every learner has the same expected performance. Therefore, inductive bias (assumptions) is necessary to achieve good generalisation on a specific problem class.

5. Loss Functions

A loss function L(y, h(x)) measures the discrepancy between the predicted value h(x) and the true label y.

5.1 0-1 Loss (Classification)

L0-1(y, h(x)) = 𝟙[h(x) ≠ y] = { 0 if h(x)=y, 1 if h(x)≠y }

Simply counts misclassifications. Non-differentiable, so not directly used in gradient-based optimisation.

5.2 Squared Error Loss (Regression)

LSE(y, h(x)) = (y − h(x))²

Penalises large errors quadratically. Differentiable and convex. Mean form (MSE):

MSE = (1/n) ∑i=1n (yi − h(xi))²

💡 Worked Example — MSE Calculation

Given predictions and actuals:

iyiŷi(yi − ŷi)²
132.50.25
254.80.04
377.30.09
422.10.01

MSE = (0.25 + 0.04 + 0.09 + 0.01) / 4 = 0.39 / 4 = 0.0975

5.3 Absolute Error Loss (Regression)

LAE(y, h(x)) = |y − h(x)|

More robust to outliers than squared error. Mean form: MAE = (1/n) ∑ |yi − h(xi)|

5.4 Cross-Entropy Loss (Classification)

LCE(y, p) = −[y·log(p) + (1−y)·log(1−p)]

Where y ∈ {0, 1} is the true label and p = P(y=1|x) is the predicted probability. This is the standard loss for logistic regression and neural network classifiers.

💡 Worked Example — Cross-Entropy

True label: y=1, Predicted: p=0.9

L = −[1·log(0.9) + 0·log(0.1)] = −log(0.9) = −(−0.1054) = 0.1054

True label: y=1, Predicted: p=0.1 (bad prediction)

L = −log(0.1) = 2.3026 (much larger penalty!)

6. Empirical Risk Minimization (ERM)

Definition — True Risk (Expected Risk)

R(h) = E(x,y)~P[L(y, h(x))] — the expected loss over the true (unknown) data distribution P.

Definition — Empirical Risk

R̂n(h) = (1/n) ∑i=1n L(yi, h(xi)) — the average loss over the training set as a proxy for R(h).

ERM Principle

Choose the hypothesis h* ∈ H that minimises the empirical risk:

h* = argminh∈H (1/n) ∑i=1n L(yi, h(xi))

Under regularity conditions (finite H, i.i.d. samples), by the law of large numbers, R̂n(h) → R(h) as n → ∞.

⚠ ERM Pitfall — Overfitting

Minimising the empirical risk alone can lead to overfitting — the model memorises training data but fails on unseen data. Solutions include:

  • Regularisation: Add a penalty term λ·||w||² to prevent large weights
  • Structural Risk Minimization (SRM): Balance empirical risk with model complexity
  • Cross-validation: Estimate generalisation error using data splits

💡 Example — ERM with Squared Loss

For linear regression with squared loss, the ERM objective becomes:

h* = argminw (1/n) ∑i=1n (yi − wTxi)²

This has the closed-form solution: w* = (XTX)−1XTy (the Normal Equation, covered in detail in the next section).

8. Generalisation and Overfitting

Definition — Generalisation

A model generalises well if its performance on unseen test data is close to its performance on training data. The generalisation gap = Test Error − Training Error.

Key factors affecting generalisation:

  • Model complexity: More complex models (higher degree polynomials, deeper trees) can fit training data better but may overfit.
  • Training set size: More data → better generalisation (up to a point).
  • Noise in data: Noisy labels make overfitting worse.
  • Regularisation: Constraining model complexity helps generalisation.

Fitting y = sin(x) + ε with polynomials of degree d:

Summary

📚 Key Takeaways

  • Supervised learning learns from labelled (x, y) pairs to predict labels for new inputs.
  • Classification → discrete output; Regression → continuous output.
  • Always split data into training, validation, and test sets.
  • The hypothesis space defines which functions the model can represent; inductive bias constrains the search.
  • Common loss functions: 0-1 loss, squared error, absolute error, cross-entropy.
  • ERM minimises average training loss; regularisation prevents overfitting.
  • No Free Lunch: no single algorithm dominates on all problems.
