GATE Previous Year Questions — Informed Search

GATE Relevance: High

Prerequisites

  • A* Search — f(n)=g(n)+h(n), optimality proof, node expansion order
  • Admissibility & Consistency — Conditions for A* optimality
  • Greedy Best-First Search — f(n)=h(n), completeness limitations
  • Heuristic Functions — Dominance, relaxed problems, pattern databases
  • Hill Climbing & SA — Local search and acceptance probability

🎯 Learning Objectives

  • Practice with authentic GATE-style problems on informed search
  • Master the step-by-step trace technique for A* expansion questions
  • Identify common traps and distractors involving admissibility, consistency, and optimality
  • Build speed and accuracy for the 2-mark MCQ and NAT formats

This page contains 10 carefully curated GATE-style questions that test the most frequently examined concepts in informed search. Each question includes a detailed solution walkthrough and a key takeaway for exam preparation.

Question 1 — A* Optimality Proof

📝 Problem Statement (MCQ — 2 marks)

Consider A* tree search with an admissible heuristic h(n). Let C* be the cost of the optimal solution. Which of the following is always true when A* selects a node n for expansion?

  1. f(n) ≤ C*
  2. f(n) = C*
  3. g(n) ≤ C*
  4. h(n) ≤ g(n)
✅ Detailed Solution

The key theorem of A* is: A* with an admissible heuristic never expands a node with f(n) > C*.

Proof sketch: Suppose A* is about to expand node n. There is always a node n′ on the optimal path in the open list with f(n′) ≤ C* (because h is admissible, so f along the optimal path never exceeds C*). Since A* picks the node with minimum f, f(n) ≤ f(n′) ≤ C*.

  • A: f(n) ≤ C* — Correct. This is the direct implication of the theorem.
  • B: f(n) = C* — Wrong. Nodes early in the search have f(n) < C*.
  • C: g(n) ≤ C* — Not always. In tree search without cycle detection, a node can be reached via a suboptimal path where g(n) > C*.
  • D: h(n) ≤ g(n) — No guarantee of this relationship.

Answer: A

Key Takeaway: The f(n) ≤ C* property is the cornerstone of A* optimality. It guarantees that A* only explores nodes that could lie on an optimal path.

Question 2 — Identifying Admissible Heuristics

📝 Problem Statement (MCQ — 1 mark)

For the 8-puzzle, which of the following heuristics is admissible?

  1. h1(n) = number of misplaced tiles
  2. h2(n) = sum of Manhattan distances of all tiles from their goal positions
  3. h3(n) = 2 × (number of misplaced tiles)
  4. Both A and B
✅ Detailed Solution

A heuristic is admissible if it never overestimates the true cost to reach the goal.

  • h1 (misplaced tiles): Each misplaced tile needs at least 1 move to reach its goal position, so h1 never overestimates. ✅ Admissible.
  • h2 (Manhattan distance): Each tile must move at least its Manhattan distance, and tiles don’t interfere (relaxation). h2 never overestimates. ✅ Admissible.
  • h3 (2 × misplaced): Consider a state where 1 tile is misplaced and adjacent to its goal — true cost = 1, h3 = 2. ❌ Overestimates.

Answer: D (Both A and B)

Key Takeaway: To verify admissibility, find the easiest scenario for each tile and check whether the heuristic could ever exceed the true cost. Multiplying an admissible heuristic by a constant > 1 usually destroys admissibility.

Question 3 — Heuristic Dominance

📝 Problem Statement (MCQ — 2 marks)

Two admissible heuristics h1 and h2 satisfy h2(n) ≥ h1(n) for all nodes n. Which statement is correct?

  1. A* with h1 always expands fewer nodes than A* with h2
  2. A* with h2 always expands fewer or equal nodes than A* with h1
  3. Both find different optimal paths
  4. h2 is not admissible since it gives larger values
✅ Detailed Solution

When h2(n) ≥ h1(n) for all n, we say h2 dominates h1. Since both are admissible, both lead to optimal solutions. However:

A* never expands nodes with f(n) > C*. With a higher (but still admissible) heuristic, f-values are closer to C*, so fewer nodes have f(n) ≤ C*. Therefore:

  • A: Wrong — backwards.
  • B: Correct — the dominating heuristic leads to fewer or equal expansions.
  • C: Wrong — both are optimal (same cost, possibly same path).
  • D: Wrong — a higher value does not violate admissibility as long as h(n) ≤ h*(n).

Answer: B

Key Takeaway: Among admissible heuristics, higher is better (more informed = fewer nodes expanded). Always prefer the dominating heuristic. If two heuristics are incomparable, use h(n) = max(h1(n), h2(n)), which is also admissible.

Question 4 — A* Node Expansion Count (NAT)

📝 Problem Statement (NAT — 2 marks)

Consider the following graph with start node S and goal node G. Edge costs are shown. The heuristic values are: h(S)=7, h(A)=6, h(B)=2, h(C)=1, h(G)=0.

    S --1--> A --4--> G
    |               ^
    2              /
    |            1
    v          /
    B --3--> C

How many nodes does A* (graph search) expand before finding the goal? Include the goal node if it gets expanded.

✅ Detailed Solution

Let’s trace A* step by step. Open list is sorted by f(n) = g(n) + h(n).

Initial: Open = {S: f=0+7=7}

Step 1: Expand S (f=7). Generate A (g=1, f=1+6=7) and B (g=2, f=2+2=4).
Open = {B:4, A:7}. Closed = {S}.

Step 2: Expand B (f=4). Generate C (g=2+3=5, f=5+1=6).
Open = {C:6, A:7}. Closed = {S, B}.

Step 3: Expand C (f=6). Generate G (g=5+1=6, f=6+0=6).
Open = {G:6, A:7}. Closed = {S, B, C}.

Step 4: Expand G (f=6). Goal test on expansion → Goal found!

Nodes expanded: S, B, C, G = 4

Path: S → B → C → G, cost = 2+3+1 = 6. (Compare: S → A → G = 1+4 = 5. Wait — let’s check.)

Actually, the path S→A→G costs 1+4=5 with f(A)=7. But S→B→C→G costs 6 with f(G)=6. Since A has f=7 > 6 = f(G), G is expanded first. But is the path via A cheaper? Yes: cost 5 < 6.

Correction: A* should find the optimal path. Let’s re-check: when G is added via C at g=6, A is still in the open list at f=7. A* expands G at f=6 first. But G is the goal — A* returns path cost 6. However, A has f=7, and expanding A would give G at g=5, f=5. This shows the heuristic h is NOT consistent (h(S)=7 but c(S,A)+h(A) = 1+6=7 = h(S) ☑; h(B)=2 but c(B,C)+h(C)=3+1=4 > h(B) ☑; h(A)=6 but c(A,G)+h(G)=4+0=4 < h(A)=6 ❌). Since h is admissible but NOT consistent, graph search A* may return a suboptimal path unless re-opening is allowed. With standard graph search (no re-opening), A* expands 4 nodes (S, B, C, G).

Answer: 4

Key Takeaway: Always check for consistency when doing A* graph search traces. With an admissible but inconsistent heuristic, graph search A* (without re-opening) may miss optimal paths. GATE often tests this subtle distinction.

Question 5 — Greedy Best-First Search Completeness

📝 Problem Statement (MCQ — 1 mark)

Which of the following is true about Greedy Best-First Search?

  1. It is complete and optimal
  2. It is complete but not optimal
  3. It is neither complete nor optimal in general
  4. It is optimal but not complete
✅ Detailed Solution

Greedy Best-First Search uses f(n) = h(n), expanding the node that appears closest to the goal.

  • Completeness: In tree search, Greedy can get stuck in infinite loops (e.g., A → B → A → B…). Even with graph search, it is only complete in finite state spaces with repeated-state checking. In general, it is not complete.
  • Optimality: Greedy ignores path cost g(n), so it can easily find a suboptimal path. Not optimal.

Answer: C

Key Takeaway: Greedy BFS = “neither complete nor optimal in general.” It is fast when the heuristic is good, but provides no guarantees. In contrast, A* is both complete and optimal (with admissible h).

Question 6 — Consistency vs Admissibility

📝 Problem Statement (MCQ — 2 marks)

Consider the following statements about heuristic functions:

  1. Every consistent heuristic is admissible.
  2. Every admissible heuristic is consistent.
  3. A* graph search is optimal with a consistent heuristic.
  4. A* tree search is optimal with an admissible heuristic.

Which statements are correct?

  1. I and II only
  2. I, III, and IV only
  3. I and III only
  4. All four
✅ Detailed Solution
  • Statement I: Consistency implies h(n) ≤ c(n,n′) + h(n′). By induction along any path to the goal, this implies h(n) ≤ h*(n), which is admissibility. ✅ True.
  • Statement II: Counter-example exists (as in Q4). An admissible heuristic need not satisfy the triangle inequality. ❌ False.
  • Statement III: With consistency, f-values are non-decreasing along any path, so the first time a node is expanded, it is via the cheapest path. No re-opening needed. ✅ True.
  • Statement IV: In tree search there is no closed list, so admissibility is sufficient for optimality. ✅ True.

Answer: B (I, III, and IV)

Key Takeaway: Consistency ⇒ Admissibility, but NOT vice versa. For graph search, you need consistency (or re-opening). For tree search, admissibility suffices. This is one of the most tested distinctions in GATE.

Question 7 — Hill Climbing Problems

📝 Problem Statement (MCQ — 1 mark)

Which of the following problems does NOT affect basic hill climbing?

  1. Local maxima
  2. Plateaux (flat regions)
  3. Ridges
  4. Exponential time complexity
✅ Detailed Solution

Hill climbing is a local search algorithm that always moves to the neighbour with the highest evaluation value (maximisation) or lowest cost (minimisation).

  • A: Local maxima — Yes, HC gets stuck at local maxima where all neighbours are worse. ✅ Affects HC.
  • B: Plateaux — Yes, when all neighbours have the same value, HC doesn’t know which way to move. ✅ Affects HC.
  • C: Ridges — Yes, ridges have slopes in multiple dimensions that are hard to navigate with single-step moves. ✅ Affects HC.
  • D: Exponential time complexity — HC runs in O(n) per step for n neighbours and terminates quickly (often in constant number of steps). It does NOT suffer from exponential time — that’s a problem of systematic search, not local search. ❌ Does NOT affect HC.

Answer: D

Key Takeaway: The three classic problems of hill climbing are: local maxima, plateaux, and ridges. Hill climbing is fast but incomplete — it trades completeness for speed.

Question 8 — IDA* Threshold Updates

📝 Problem Statement (MCQ — 2 marks)

IDA* performs iterative deepening with f-cost limits. In the first iteration, the threshold is f(start) = g(start) + h(start) = h(start). Suppose during the first iteration, the following f-values exceed the threshold: 12, 15, 14, 18, 13. The current threshold is 10. What is the next threshold?

  1. 15 (the median)
  2. 12 (the minimum exceeding value)
  3. 18 (the maximum)
  4. 14.4 (the average)
✅ Detailed Solution

IDA* sets the next threshold to the minimum f-value that exceeded the current threshold. This ensures we explore all nodes with f ≤ new threshold in the next iteration.

Exceeded values: {12, 15, 14, 18, 13}. Minimum = 12.

Answer: B (12)

Key Takeaway: IDA* threshold update rule: next_threshold = min(all f-values that exceeded current threshold). This is analogous to how iterative deepening increases the depth limit by 1 in IDDFS, but here it increases to the next f-value “contour.”

Question 9 — Simulated Annealing Acceptance Probability

📝 Problem Statement (NAT — 2 marks)

In simulated annealing for a minimisation problem, the current state has cost 50 and a neighbour has cost 55. The temperature T = 10. What is the probability of accepting the worse neighbour? Round to two decimal places.

Hint: P = e−ΔE/T where ΔE = new_cost − current_cost

✅ Detailed Solution

ΔE = 55 − 50 = 5 (positive, since the new state is worse)

P = e−5/10 = e−0.5 ≈ 0.61

So there is a 61% chance of accepting this worse move at T=10.

Answer: 0.61

Key Takeaway: SA acceptance probability: P = e−ΔE/T. As T decreases, the probability of accepting worse moves drops toward 0. As T → ∞, P → 1 (random walk). As T → 0, P → 0 (pure hill climbing). In GATE, you may need to compute e−x for small x values.

Question 10 — A* with Multiple Heuristics

📝 Problem Statement (MCQ — 2 marks)

Given two admissible heuristics h1 and h2 that are not comparable (neither dominates the other), you define h3(n) = max(h1(n), h2(n)). Which of the following is true?

  1. h3 is admissible and dominates both h1 and h2
  2. h3 is not admissible
  3. h3 is admissible but does not dominate either h1 or h2
  4. h3 may or may not be admissible depending on the problem
✅ Detailed Solution

Admissibility: For any node n, h1(n) ≤ h*(n) and h2(n) ≤ h*(n). Therefore max(h1(n), h2(n)) ≤ h*(n). So h3 is admissible. ✅

Dominance: h3(n) = max(h1(n), h2(n)) ≥ h1(n) and ≥ h2(n) for all n. So h3 dominates both. ✅

Answer: A

Key Takeaway: The max of admissible heuristics is always admissible and dominates all constituent heuristics. This is a powerful technique: if you have multiple heuristics for a problem, use their pointwise maximum. This is commonly tested in GATE.

Exam Strategy Tips

  • Trace methodically: For A* expansion questions, maintain an explicit open list sorted by f(n) on paper. Show g, h, and f for each node.
  • Check consistency: If a question involves A* graph search, immediately check whether h is consistent. If not, note that re-opening may be needed.
  • Admissibility verification: To prove h is admissible, show it corresponds to a relaxed problem. To disprove, find ONE counter-example where h(n) > h*(n).
  • Dominance: Given multiple heuristics, the one with the highest values (while remaining admissible) leads to the fewest node expansions.
  • SA formula: Memorise P = e−ΔE/T. Remember ΔE is positive for worse moves. Know that e−0.5 ≈ 0.61, e−1 ≈ 0.37, e−2 ≈ 0.14.
  • IDA* threshold: Next threshold = minimum f-value that exceeded the current threshold.
