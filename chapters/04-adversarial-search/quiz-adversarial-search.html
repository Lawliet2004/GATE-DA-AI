Quiz: Adversarial Search

15 Questions

Test your understanding of minimax, alpha-beta pruning, MCTS, evaluation functions, and game theory. Mix of MCQ, MSQ, NAT, and True/False questions.

Q1. (MCQ) What is the time complexity of the minimax algorithm for a game tree with branching factor b and depth m?

Answer: B. Minimax performs a complete DFS of the game tree. Each of the m levels has branching factor b, so the total number of leaf nodes is bm, and all must be visited.

Q2. (MCQ) Consider a game tree: MAX at root with two MIN children. Left MIN has leaves {7, 3}. Right MIN has leaves {5, 8}. What is the minimax value at the root?

Answer: C. Left MIN = min(7,3) = 3. Right MIN = min(5,8) = 5. Root MAX = max(3,5) = 5.

Q3. (True/False) Alpha-beta pruning always returns the same result as the full minimax algorithm.

Answer: True. Alpha-beta pruning only removes branches that provably cannot affect the final minimax value. The root value is always identical to full minimax.

Q4. (MCQ) In alpha-beta pruning, α represents:

Answer: B. α is the best (highest) value that MAX can guarantee so far along the path from the root. β is the best (lowest) value MIN can guarantee.

Q5. (NAT) A game tree has branching factor 4 and depth 4. In the best case of alpha-beta pruning, how many leaf nodes are evaluated?

Use the formula: 2bd/2 − 1 for even d.

Answer: 31. Best case for even d: 2 × bd/2 − 1 = 2 × 42 − 1 = 2 × 16 − 1 = 31. Total without pruning: 44 = 256.

Q6. (MCQ) Which game is NOT a zero-sum game?

Answer: D. Prisoner’s Dilemma is a general-sum game where both players can lose (−2, −2) or gain asymmetrically. The payoffs don’t always sum to zero. Chess, Tic-Tac-Toe, and Go are zero-sum.

Q7. (True/False) The effectiveness of alpha-beta pruning is independent of the order in which moves are examined.

Answer: False. Move ordering critically affects pruning. With perfect ordering, complexity is O(bd/2). With worst ordering, no pruning occurs and complexity is O(bd).

Q8. (MCQ) In a stochastic game tree, the value at a CHANCE node is computed as:

Answer: C. CHANCE node value = ∑ P(outcome) × value(outcome). This is the expected value, computed as a probability-weighted average.

Q9. (NAT) A chance node has two outcomes: outcome A (probability 0.5, value 5) and outcome B (probability 0.5, value 2). What is the value at the chance node?

Answer: 3.5. Value = 0.5 × 5 + 0.5 × 2 = 2.5 + 1.0 = 3.5.

Q10. (MCQ) The four phases of MCTS in order are:

Answer: A. The correct order is: (1) Selection — traverse tree using UCB1, (2) Expansion — add new child node, (3) Simulation — random playout to terminal state, (4) Backpropagation — update stats from leaf to root.

Q11. (MSQ) Which of the following are techniques to improve move ordering in alpha-beta search? (Select all that apply)

Answer: A, B, D. Killer moves, transposition tables, and iterative deepening all improve move ordering. Increasing the branching factor would make the problem harder, not improve ordering.

Q12. (True/False) The minimax value of Tic-Tac-Toe from the initial state is 0 (draw).

Answer: True. With perfect play by both sides, Tic-Tac-Toe always ends in a draw. The minimax value of the initial position is 0.

Q13. (MCQ) The “horizon effect” in game-playing programs refers to:

Answer: B. The horizon effect occurs when important events (captures, threats) lie just beyond the search depth cutoff. The engine cannot see them and may make delaying moves that worsen its position. Quiescence search is the standard remedy.

Q14. (MCQ) In the UCB1 formula used in MCTS, the exploration constant C controls:

Answer: C. UCB1 = wi/ni + C√(ln N / ni). The first term is exploitation (high win rate). The second term, scaled by C, is exploration (less visited). Higher C means more exploration; lower C means more exploitation.

Q15. (NAT) In the following game tree, apply alpha-beta pruning (left-to-right). MAX root with two MIN children. Left MIN has leaves {6, 4, 9}. Right MIN has leaves {2, 8, 5}. What is the minimax value at the root?

Answer: 4. Left MIN = min(6, 4, 9) = 4. Back at MAX: α = 4. Right MIN: first child = 2. Since 2 ≤ α(4), α cutoff! Prune 8 and 5. Root MAX = max(4, 2) = 4.

Q16. (True/False) MCTS requires a hand-crafted evaluation function to work.

Answer: False. One of MCTS’s key advantages is that it does NOT require an evaluation function. Instead, it uses random playouts (simulations) to estimate position values. However, adding a learned evaluation (as in AlphaGo) can dramatically improve performance.
