Partially Observable Games

GATE Relevance: Low

Prerequisites

  • Stochastic Games — Chance nodes and expectiminimax
  • Game Theory Basics — Perfect vs. imperfect information

🎯 Learning Objectives

  • Distinguish between perfect and imperfect information games
  • Understand the concept of belief states in games
  • Analyse simple card game scenarios with hidden information
  • Know the approaches to imperfect information: determinization, counterfactual regret

1. Imperfect Information Games

In imperfect information games, players do not have full knowledge of the current game state. Some information is hidden from one or both players.

Imperfect Information

A game has imperfect information if at least one player, at some point during the game, does not know the complete state. The unknown portion may include:

  • Opponent’s cards (Poker, Bridge)
  • Hidden pieces (Battleship, Stratego)
  • Opponent’s previous actions (some simultaneous-move games)
PropertyPerfect InformationImperfect Information
State knowledgeBoth players see complete stateSome state information is hidden
ExamplesChess, Go, Tic-Tac-ToePoker, Bridge, Battleship
AlgorithmMinimax / Alpha-BetaBelief-state search, CFR
ComplexityGame tree sizeInformation set size × game tree

2. Belief States

Belief State

A belief state is a probability distribution over all possible actual game states, given a player’s observations. It represents what the player believes about the hidden information.

Poker Belief State Example

In Texas Hold’em, you see your 2 hole cards and the community cards. Your belief state is a distribution over the possible hands your opponent could hold:

Your cards: A♠ K♠
Community: 10♠ J♠ 2♥ 5♣ (turn)
Cards remaining in deck: 46

Possible opponent hands: C(46, 2) = 1035 combinations
Each is equally likely a priori.

But: opponent raised pre-flop, suggesting strong cards.
Updated belief: higher probability on premium hands
  (AA, KK, QQ, AK, AQ, etc.)

The player must maintain and update this belief state as new information becomes available (community cards revealed, opponent actions observed).

3. Information Sets

Information Set

An information set for a player is a set of game tree nodes that the player cannot distinguish between. At all nodes in the same information set, the player must choose the same strategy (since they don’t know which node they’re at).

Simple Card Game (Kuhn Poker)

3 cards: Jack, Queen, King. Each player gets one card.

Player 1 holds Jack:
  Player 2 could have Queen or King
  → Information set: {(J vs Q), (J vs K)}

Player 1 holds Queen:  
  Player 2 could have Jack or King
  → Information set: {(Q vs J), (Q vs K)}

Player 1 must choose the SAME action for
all states in their information set.

4. Approaches to Imperfect Information Games

4.1 Determinization (Monte Carlo Sampling)

Sample multiple possible “worlds” (specific assignments of hidden information), solve each as a perfect-information game, then average the results.

Determinization Approach
function DETERMINIZED-SEARCH(state, N):
    action_scores ← empty map
    for i = 1 to N:
        world ← SAMPLE-HIDDEN-INFO(state)  // Fix hidden cards
        best_action ← MINIMAX(world)         // Solve as perfect info
        action_scores[best_action] += 1
    return argmax(action_scores)
⚠ Strategy Fusion Problem

Determinization has a fundamental flaw: it may recommend different actions in different sampled worlds, producing an impossible strategy that requires knowing the hidden information. This is called strategy fusion. The approach works well for Bridge but poorly for Poker.

4.2 Counterfactual Regret Minimization (CFR)

The state-of-the-art for imperfect information games. CFR iteratively self-plays, tracking regret for not having played each action, and adjusting strategy to minimize cumulative regret.

  • Converges to Nash equilibrium in two-player zero-sum games
  • Used by Libratus/Pluribus (Carnegie Mellon) to defeat professional poker players
  • Works on information sets rather than individual game states

4.3 Comparison of Approaches

ApproachStrengthsWeaknesses
DeterminizationSimple, works for BridgeStrategy fusion; non-optimal for bluffing games
CFROptimal for zero-sum; handles bluffingComputationally expensive; convergence can be slow
MCTS + Information SetsPractical; handles large gamesApproximate; no convergence guarantee

5. Poker: AI’s Greatest Challenge

Poker exemplifies imperfect information because of hidden cards and the critical role of bluffing:

  • Heads-up no-limit Texas Hold’em: ~10161 information sets
  • Libratus (2017): First AI to beat professional poker players using CFR + depth-limited solving
  • Pluribus (2019): Extended to multiplayer poker (6 players)
Why Bluffing is Rational

In the Nash equilibrium strategy for poker, bluffing occurs with a mathematically precise frequency. If you never bluff, opponents fold to your bets only when you have strong hands, reducing your winnings. Bluffing makes your betting pattern unpredictable, forcing opponents to sometimes call with weaker hands.

Summary

  • Imperfect information games have hidden state (opponent cards, pieces, etc.).
  • Players reason using belief states (probability distributions over possible true states).
  • Information sets group indistinguishable game positions; the same strategy applies to all.
  • Determinization is simple but suffers from strategy fusion.
  • CFR converges to Nash equilibrium and handles bluffing correctly.
  • AI has surpassed human experts in poker (Libratus, Pluribus) using CFR-based approaches.
