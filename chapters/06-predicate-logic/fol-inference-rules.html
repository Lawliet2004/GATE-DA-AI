FOL Inference Rules

GATE Relevance: High

Prerequisites

  • FOL Syntax — terms, WFFs, quantifiers
  • Quantifier Rules — negation, distribution
  • Propositional Inference — Modus Ponens, Resolution

🎯 Learning Objectives

  • Apply Universal Instantiation (UI) and Universal Generalization (UG)
  • Apply Existential Instantiation (EI) and Existential Generalization (EG)
  • Combine quantifier rules with propositional inference
  • Identify soundness pitfalls with incorrect generalization/instantiation

1. Universal Instantiation (UI)

Universal Instantiation
∀x α(x)   ⊢   α(t)

From ∀x α, infer α with x replaced by any ground term t (or any term free for x in α).

Restriction: t must be free for x in α — substituting t for x must not cause any variable in t to become bound.

Example

Given: ∀x (Human(x) → Mortal(x))

UI with x = Socrates: Human(Socrates) → Mortal(Socrates)

UI with x = Father(John): Human(Father(John)) → Mortal(Father(John))

2. Universal Generalization (UG)

Universal Generalization
α(x)   ⊢   ∀x α(x)

If α(x) has been derived for an arbitrary x (not a specific individual), we can conclude ∀x α(x).

Restrictions:

  • x must not appear in any undischarged assumption.
  • x must not have been introduced by Existential Instantiation.
Soundness Pitfall

We cannot generalize over a specific constant. From Mortal(Socrates) we cannot conclude ∀x Mortal(x). Socrates is a specific individual, not arbitrary.

3. Existential Instantiation (EI)

Existential Instantiation
∃x α(x)   ⊢   α(c)

From ∃x α(x), introduce a new Skolem constant c (not appearing elsewhere in the knowledge base) and conclude α(c).

Restriction: c must be completely fresh — it has not appeared in any prior formula.

Example

Given: ∃x (Dog(x) ∧ Black(x))

EI: Dog(c1) ∧ Black(c1)   where c1 is a fresh constant.

We know some black dog exists; c1 names it, but we know nothing else about c1.

4. Existential Generalization (EG)

Existential Generalization
α(t)   ⊢   ∃x α(x)

If we know α holds for some specific term t, we can conclude ∃x α(x) (replacing t by x).

Example

Given: Loves(John, Mary)

EG: ∃x Loves(x, Mary)   (“someone loves Mary”)

EG: ∃y Loves(John, y)   (“John loves someone”)

EG: ∃x ∃y Loves(x, y)   (“someone loves someone”)

5. Summary of Four Rules

RuleFromConcludeKey Restriction
UI∀x α(x)α(t) for any tt free for x in α
UGα(x) for arbitrary x∀x α(x)x not from EI, not in assumptions
EI∃x α(x)α(c) for fresh cc must be a brand-new constant

6. Combining with Propositional Rules — Worked Example

Proof: Socrates is Mortal

Knowledge Base:

  1. ∀x (Human(x) → Mortal(x))   [All humans are mortal]
  2. Human(Socrates)   [Socrates is human]

Derivation:

  1. Human(Socrates) → Mortal(Socrates)   [UI on (1) with x = Socrates]
  2. Mortal(Socrates)   [Modus Ponens on (2) and (3)]
Proof: Someone friendly passes

KB:

  1. ∀x (Friendly(x) → Passes(x))
  2. ∃x Friendly(x)

Derivation:

  1. Friendly(c)   [EI on (2), c is fresh]
  2. Friendly(c) → Passes(c)   [UI on (1) with x = c]
  3. Passes(c)   [Modus Ponens on (3) and (4)]
  4. ∃x Passes(x)   [EG on (5)]

∴ Someone passes the exam. □

7. Soundness Concerns

Soundness of FOL Inference

An inference system is sound if every formula it derives from a set of premises is a logical consequence of those premises. The four quantifier rules, when used with their restrictions, are sound.

Unsound Derivation Example
  1. ∃x Rich(x)   [Some person is rich]
  2. Rich(c)   [EI, c is fresh]
  3. ∀x Rich(x)   [WRONG! UG on (2) — but c came from EI!]

This “proves” everyone is rich from the premise that someone is rich. The error: UG cannot be applied to a variable/constant introduced by EI.

Summary

  • UI: ∀x α(x) ⊢ α(t) for any suitable term t.
  • UG: α(x) ⊢ ∀x α(x) only when x is truly arbitrary.
  • EI: ∃x α(x) ⊢ α(c) with a fresh constant c.
  • EG: α(t) ⊢ ∃x α(x) always valid.
  • These rules bridge quantified formulas and propositional reasoning (Modus Ponens, etc.).
