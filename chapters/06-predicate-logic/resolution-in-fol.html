Resolution in First-Order Logic

GATE Relevance: High

Prerequisites

  • Unification — MGU, substitution
  • Quantifier Rules — prenex normal form, negation
  • Propositional Resolution

🎯 Learning Objectives

  • Convert any FOL sentence to clausal (CNF) form
  • Perform Skolemization (replacing ∃ with Skolem constants/functions)
  • Apply the resolution rule with unification
  • Prove statements by refutation (resolution-refutation)

1. Converting to Clausal Form

To use resolution in FOL, every formula must be converted to clausal form (a conjunction of clauses, each clause being a disjunction of literals). The 7-step process:

Steps to Clausal Form
  1. Eliminate ↔: α ↔ β  ⇒  (α → β) ∧ (β → α)
  2. Eliminate →: α → β  ⇒  ¬α ∨ β
  3. Move ¬ inward (using De Morgan’s + quantifier negation):
    • ¬(α ∧ β) ⇒ ¬α ∨ ¬β
    • ¬(α ∨ β) ⇒ ¬α ∧ ¬β
    • ¬∀x α ⇒ ∃x ¬α
    • ¬∃x α ⇒ ∀x ¬α
    • ¬¬α ⇒ α
  4. Standardize variables: rename so each quantifier uses a unique variable.
  5. Skolemize: remove ∃ quantifiers (see below).
  6. Drop ∀: all remaining variables are implicitly universally quantified.
  7. Distribute ∨ over ∧: α ∨ (β ∧ γ) ⇒ (α ∨ β) ∧ (α ∨ γ)

2. Skolemization

Skolemization Rules

For each existentially quantified variable:

  • If ∃x appears with no universally quantified variables in scope, replace x with a Skolem constant (e.g., c, A, sk0).
  • If ∃x appears within the scope of ∀y1…∀yk, replace x with a Skolem function f(y1,…,yk).

Then delete the ∃x quantifier.

Skolemization Examples
  • ∃x P(x) ⇒ P(c)   [Skolem constant c]
  • ∀x ∃y Loves(x,y) ⇒ ∀x Loves(x, f(x))   [Skolem function f]
  • ∀x ∀y ∃z Between(x,y,z) ⇒ ∀x ∀y Between(x, y, g(x,y))   [Skolem function g(x,y)]
Key Point

Skolemization preserves satisfiability but does NOT preserve logical equivalence. The original and Skolemized formulas are equisatisfiable: one is satisfiable iff the other is. For resolution-refutation proofs, this suffices.

3. The Resolution Rule with Unification

First-Order Resolution

Given two clauses C1 and C2 (with no shared variables — standardize apart first):

If literal l1 ∈ C1 and literal ¬l2 ∈ C2 (or l2 ∈ C2 and ¬l1 ∈ C1), and θ = MGU(l1, l2), then the resolvent is:

(C1θ − {l1θ}) ∪ (C2θ − {¬l2θ})
Resolution Step Example

C1: ¬Human(x) ∨ Mortal(x)   [from ∀x (Human(x)→Mortal(x))]

C2: Human(Socrates)

Unify Human(x) with Human(Socrates): θ = {x/Socrates}

Resolvent: Mortal(Socrates) ✓

4. Proof by Refutation

Resolution-Refutation Procedure

To prove KB ⊨ α:

  1. Add ¬α to the knowledge base (negate the goal).
  2. Convert everything to clausal form.
  3. Repeatedly apply resolution to derive new clauses.
  4. If the empty clause □ is derived, then KB ⊨ α.
Refutation Completeness

First-order resolution is refutation complete: if a set of clauses is unsatisfiable, resolution will eventually derive the empty clause. (However, if the set is satisfiable, the procedure may run forever — reflecting the semi-decidability of FOL.)

5. Full Worked Example: Proof by Resolution

Prove: “Fido is mortal”

KB:

  1. ∀x (Dog(x) → Animal(x))   [All dogs are animals]
  2. ∀x (Animal(x) → Mortal(x))   [All animals are mortal]
  3. Dog(Fido)   [Fido is a dog]

Goal: Mortal(Fido)

Step 1: Negate goal ⇒ ¬Mortal(Fido)

Step 2: Convert to clauses:

  • C1: ¬Dog(x) ∨ Animal(x)
  • C2: ¬Animal(y) ∨ Mortal(y)
  • C3: Dog(Fido)
  • C4: ¬Mortal(Fido)   [negated goal]

Step 3: Resolution steps:

  1. Resolve C2 and C4: unify Mortal(y) with Mortal(Fido), θ={y/Fido} ⇒ C5: ¬Animal(Fido)
  2. Resolve C1 and C5: unify Animal(x) with Animal(Fido), θ={x/Fido} ⇒ C6: ¬Dog(Fido)
  3. Resolve C3 and C6: unify Dog(Fido) with Dog(Fido), θ={} ⇒ □ (empty clause)

Empty clause derived ⇒ Mortal(Fido) is proved. □

Summary

  • Clausal form conversion: eliminate →/↔, push ¬ in, standardize, Skolemize, drop ∀, distribute.
  • Skolemization replaces ∃ with constants (no ∀ scope) or functions (with ∀ scope).
  • Resolution in FOL = propositional resolution + unification.
  • Proof by refutation: negate the goal, derive empty clause □.
  • Resolution is refutation-complete for FOL.
